## Technical Implementation Details

### Custom Tailwind Pagination Helper

```ruby
def pagy_tailwind_nav(pagy)
  html = +%(<nav class="flex items-center justify-center gap-1" aria-label="Pagination">)

  # Previous button
  if pagy.prev
    html << link_to("←", pagy_url_for(pagy, pagy.prev),
                   class: "px-3 py-2 text-sm font-medium text-gray-700 bg-white border border-gray-300 rounded-md hover:bg-gray-50",
                   aria_label: "Previous")
  else
    html << %(<span class="px-3 py-2 text-sm font-medium text-gray-400 bg-gray-100 border border-gray-300 rounded-md cursor-not-allowed">←</span>)
  end

  # Page numbers with active state styling
  pagy.series.each do |item|
    case item
    when Integer
      if item == pagy.page
        html << %(<span class="px-3 py-2 text-sm font-medium text-blue-600 bg-blue-50 border border-blue-300 rounded-md">#{item}</span>)
      else
        html << link_to(item, pagy_url_for(pagy, item),
                       class: "px-3 py-2 text-sm font-medium text-gray-700 bg-white border border-gray-300 rounded-md hover:bg-gray-50")
      end
    when "gap"
      html << %(<span class="px-2 py-2 text-sm text-gray-400">...</span>)
    end
  end

  # Next button
  if pagy.next
    html << link_to("→", pagy_url_for(pagy, pagy.next),
                   class: "px-3 py-2 text-sm font-medium text-gray-700 bg-white border border-gray-300 rounded-md hover:bg-gray-50",
                   aria_label: "Next")
  else
    html << %(<span class="px-3 py-2 text-sm font-medium text-gray-400 bg-gray-100 border border-gray-300 rounded-md cursor-not-allowed">→</span>)
  end

  html << %(</nav>)
  html.html_safe
end
```

### Expandable Controller (Stimulus)

```javascript
import { Controller } from "@hotwired/stimulus"

export default class extends Controller {
  static targets = ["content", "toggle"]
  static values = { lines: { type: Number, default: 3 } }

  connect() {
    this.checkHeight()
  }

  checkHeight() {
    const lineHeight = parseFloat(getComputedStyle(this.contentTarget).lineHeight)
    const maxHeight = lineHeight * this.linesValue
    const actualHeight = this.contentTarget.scrollHeight

    if (actualHeight > maxHeight) {
      this.contentTarget.style.maxHeight = `${maxHeight}px`
      this.contentTarget.style.overflow = "hidden"
      this.toggleTarget.classList.remove("hidden")
    } else {
      this.toggleTarget.classList.add("hidden")
    }
  }

  toggle() {
    const isExpanded = this.contentTarget.style.maxHeight === "none"

    if (isExpanded) {
      // Collapse
      const lineHeight = parseFloat(getComputedStyle(this.contentTarget).lineHeight)
      const maxHeight = lineHeight * this.linesValue
      this.contentTarget.style.maxHeight = `${maxHeight}px`
      this.contentTarget.style.overflow = "hidden"
      this.toggleTarget.textContent = "More..."
    } else {
      // Expand
      this.contentTarget.style.maxHeight = "none"
      this.contentTarget.style.overflow = "visible"
      this.toggleTarget.textContent = "Less"
    }
  }
}
```

## Design Decisions & Learnings

### Why Card-Based Design?
**Decision:** Individual cards with shadows instead of list with dividers

**Reasoning:**
- Better visual separation between commits
- More modern, clean appearance
- Room for expansion (tags, actions, etc.)
- Easier to scan at a glance

**User Feedback:**
- "This is even harder to read" (after tight spacing)
- "The pagination is not styled that great"
- "There's still not enough space between commit rows"
- Led to multiple iterations until spacing was right

### Why Two-Column Layout?
**Decision:** Summary and Details side-by-side horizontally

**Reasoning:**
- Better use of horizontal space
- Reduces vertical scrolling
- Clear visual separation of concerns
- Professional dashboard appearance

**Challenge:**
- Grid layout initially collapsed (both columns rendering conditionally)
- Fixed by always rendering both column divs, conditionally showing content inside

### Why Stimulus for Expandability?
**Decision:** Use Stimulus controller instead of vanilla JS or jQuery

**Reasoning:**
- Already part of Rails 8 stack (no new dependencies)
- Clean, reusable controller pattern
- Auto-loads without manual registration
- Easy to test and maintain

## Issues Encountered & Solutions

### Issue 1: Tailwind Pagination Not Working
**Problem:** Tried to use `pagy/extras/tailwind` which doesn't exist in Pagy 9.4.0

**Solution:** Created custom `pagy_tailwind_nav` helper with manual Tailwind styling

### Issue 2: Newlines Not Preserved
**Problem:** `whitespace-pre-line` Tailwind class wasn't working

**Solution:** Used inline CSS `white-space: pre-line` (later moved to CSS file)

### Issue 3: Grid Columns Collapsing
**Problem:** Two-column grid showed columns vertically stacked

**Root Cause:** Conditional rendering of column divs meant only one existed at a time

**Solution:** Always render both column divs, conditionally show content inside

### Issue 4: Multiple Design Iterations
**Problem:** Initial layouts were "hard to read" and "harder to read"

**User Feedback Driven Iteration:**
1. Separated title from details → Still dense
2. Added card styling → Still cramped (no spacing)
3. Added spacing between cards → Columns still stacked
4. Fixed grid layout → Success! ✅

**Lesson:** User feedback is critical for UI/UX. What seems good to developer may not work for users.
